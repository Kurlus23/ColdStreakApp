# ColdStreak Watch — Xcode Integration Guide

This folder contains the full source for the watchOS app plus the iOS-side Capacitor bridge plugin. Capacitor doesn't support watchOS directly, so the watch app is a separate native target inside the same Xcode project as the iOS app.

> **Targets:** watchOS 10.0+, paired iPhone running iOS 17.0+

---

## Step 1 — Run `npx cap sync ios` first

If the iOS project isn't already generated locally, run this from the repo root on your Mac:

```bash
npx cap sync ios
open ios/App/App.xcworkspace
```

You should now see the existing `App` iOS target inside Xcode.

---

## Step 2 — Add a new Watch App target

1. In Xcode: **File → New → Target…**
2. Choose **watchOS → App**, click **Next**.
3. Fill in:
   - Product Name: `ColdStreakWatch`
   - Bundle Identifier: `<your iOS bundle id>.watchkitapp` (e.g. `com.coldstreak.app.watchkitapp`)
   - Interface: **SwiftUI**
   - Language: **Swift**
   - **Uncheck** "Include Notification Scene"
   - **Uncheck** "Include Tests"
4. Click **Finish**. Xcode will scaffold a `ColdStreakWatch Watch App` group and auto-pair it to your iOS app target.

---

## Step 3 — Replace the auto-generated files

1. In the new `ColdStreakWatch Watch App` group, **delete** the placeholder files Xcode created (`ColdStreakWatchApp.swift`, `ContentView.swift`, etc.). Choose **Move to Trash**.
2. **Drag** the entire contents of this repo's `watchos/ColdStreakWatch/` folder into that group in the Xcode navigator.
   - In the import dialog: ✓ "Copy items if needed", ✓ "Create groups", and **only** check the `ColdStreakWatch Watch App` target (NOT the iOS app).
3. **Replace `Info.plist`**: in the watch target's Build Settings, point "Info.plist File" at the file you just dragged in (`ColdStreakWatch/Resources/Info.plist`), or simply delete the autogenerated one and use this.
4. **Set deployment target**: select the watch target → **General → Minimum Deployments → watchOS 10.0**.

---

## Step 4 — Wire up HealthKit entitlements (watch target)

1. Select the **ColdStreakWatch Watch App** target → **Signing & Capabilities**.
2. Click **+ Capability** → add **HealthKit**.
3. Then under "Code Signing Entitlements" point at `ColdStreakWatch/Resources/ColdStreakWatch.entitlements` (or let Xcode write its own — both work; the file in this repo is a reference).
4. Make sure **"Background Modes"** isn't needed manually — `WKBackgroundModes = workout-processing` is already in the Info.plist.

---

## Step 5 — Add the iOS-side bridge plugins

The watch sends each completed plunge to the iPhone via `WatchConnectivity`. The iPhone also pulls HR/HRV from Apple Health for plunges logged directly on the phone. Both pieces are Capacitor plugins.

### 5a — WatchSyncPlugin (receives plunges from the watch)

1. In Xcode, select the **iOS App** target group (`App/App/`).
2. Drag `watchos/ios-bridge/WatchSyncPlugin.swift` into it. In the import dialog: ✓ "Copy items if needed", ✓ "Create groups", and check the **App** target only (NOT the watch target).
3. No additional registration needed — Capacitor 5+ auto-registers any `@objc(...)` annotated plugin at launch.

### 5b — HealthKitPlugin (lets the iPhone enrich plunges with HR/HRV from Apple Health)

This covers users whose tracker syncs to Apple Health but isn't an Apple Watch — Garmin, Whoop, Fitbit, Oura, T-Rex via 3rd-party sync apps, etc.

1. Drag `watchos/ios-bridge/HealthKitPlugin.swift` into the same iOS **App** target group.
2. Select the **App** target → **Signing & Capabilities** → **+ Capability** → add **HealthKit**.
3. Open the iOS app's `Info.plist` and add this row (right-click → Add Row):
   - **Key:** `NSHealthShareUsageDescription`
   - **Type:** String
   - **Value:** `ColdStreak reads heart rate and HRV during your plunges to estimate strain and recovery.`

### JS side — already wired up

The JS bridges are already in the repo:

- `client/src/lib/watchSync.ts` — drains the watch queue and POSTs each plunge to `/api/plunges`
- `client/src/lib/healthKit.ts` — pulls HR/HRV from Apple Health
- `client/src/pages/Home.tsx` — calls them on app mount and inside `doLogPlunge` as a fallback when no live BLE strap is connected

No further React work required. Just rebuild the iOS app after dragging the two `.swift` files in:

```bash
npm run build
npx cap sync ios
```

## Step 6 — App Icons

The watch needs its own icon set (different sizes from iOS). In `ColdStreakWatch/Assets.xcassets/AppIcon.appiconset/`, drop in PNGs at:
- 48×48, 55×55, 58×58, 87×87, 80×80, 88×88, 100×100, 172×172, 196×196, 216×216, 234×234, 258×258, 1024×1024

Easiest path: use a tool like https://easyappicon.com/ or Bakery to bulk-generate from a single 1024×1024 source.

## Step 7 — Build & Test

1. Plug in your iPhone with a paired Apple Watch (or use the simulator: Xcode → Window → Devices and Simulators → pair a Watch sim with an iPhone sim).
2. Select the **ColdStreakWatch Watch App** scheme + your watch destination.
3. **Cmd+R**.

### Test checklist
- [ ] On first launch, watch prompts for HealthKit permission. Grant all.
- [ ] Crown spins to adjust temp (28–70°F).
- [ ] "Start Plunge" begins the workout — Now Playing-style activity appears.
- [ ] Live HR updates within ~5 seconds.
- [ ] Haptic at 30s, 1m, 2m, 3m, 4m, 5m, then every minute.
- [ ] Always-On display: cover the watch with your sleeve mid-plunge — timer should still be visible (dimmed) when raised.
- [ ] "Stop" → moves to Recovery view, HR continues to update.
- [ ] After 3 min recovery (or "Done"), summary shows duration, max/min HR, HRV before/after.
- [ ] "Save & Sync" — payload arrives on iPhone (set a breakpoint in `WatchSyncPlugin.session(_:didReceiveUserInfo:)` to confirm).
- [ ] Open the iPhone app — the plunge appears in the user's history.
- [ ] Apple Health → "Workouts" — the plunge is logged as an "Other" workout and contributes to Move/Exercise rings.

### Common gotchas
- **HR shows `—` forever**: HealthKit permissions weren't granted, or the simulator doesn't generate HR data — test on real hardware.
- **Watch payload never arrives on phone**: confirm `WCSession.isSupported() && WCSession.default.isPaired` on both ends. The watch app must be installed (not just running in simulator).
- **Always-On not dimming**: requires running on a Series 5 or later. The simulator's Always-On debug mode is under **Features → Always On Display**.
- **"workout-processing" entitlement error**: make sure the watch target's signing team is set and Background Modes capability isn't *also* added in Xcode (Info.plist already declares it).

## Architecture summary

```
Watch                                iPhone (Capacitor app)
─────                                ─────────────────────
ReadyView ──► tap Start
              │
              ├─► HRVService.recentAverageHRV()  (last 5 min from HealthKit)
              ├─► HRVService.mostRecentRestingHR()
              └─► PlungeWorkoutManager.start()
                    │
                    ├─► HKWorkoutSession (.other / .outdoor)
                    ├─► HKLiveWorkoutBuilder streams HR → PlungeSession
                    └─► Always-On display enabled automatically

PlungeView (live timer + HR)         (idle, can be closed)
   │ haptic at 30s/1m/2m/3m…
   │
   └─► tap Stop ─► ends workout ─► RecoveryView (3 min)
                                       │
                                       └─► HRVService.recentAverageHRV() again
                                              │
                                              └─► SummaryView ─► Save & Sync
                                                                     │
                                                                     ▼
                                                         WCSession.transferUserInfo
                                                                     │
                                                                     ▼
                                                         WatchSyncPlugin
                                                         (queues + notifies JS)
                                                                     │
                                                                     ▼
                                                         POST /api/plunges
                                                         (uses existing JWT auth)
```

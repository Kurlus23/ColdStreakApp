1. Confirm your initialization is correct
Your setup sounds mostly right, but there are two things to check:

Env variable naming: For Vite-based React apps, the recommended prefix is VITE_PUBLIC_ — not just VITE_. So your .env should look like:

VITE_PUBLIC_POSTHOG_KEY=phc_...  
VITE_PUBLIC_POSTHOG_HOST=https://us.i.posthog.com  
And in analytics.ts:

import.meta.env.VITE_PUBLIC_POSTHOG_KEY  
import.meta.env.VITE_PUBLIC_POSTHOG_HOST  
Use PostHogProvider + defaults: The recommended pattern for Vite is to wrap your app with PostHogProvider and pass a defaults option, which automatically configures PostHog with recommended settings for new projects. Your main.tsx should look like:

import posthog from 'posthog-js'  
import { PostHogProvider } from 'posthog-js/react'

const options = {  
  api_host: import.meta.env.VITE_PUBLIC_POSTHOG_HOST,  
  defaults: '2026-01-30',  
} as const

posthog.init(import.meta.env.VITE_PUBLIC_POSTHOG_KEY, options)

createRoot(document.getElementById('root')!).render(  
  <StrictMode>  
    <PostHogProvider client={posthog}>  
      <App />  
    </PostHogProvider>  
  </StrictMode>  
)
